# 市场图谱 UI 优化设计规格说明书 (2026-06-23)

本规格说明书定义了市场图谱界面的视觉体验与功能优化设计，旨在去除冗余元素（Emoji 等）、统一系统 VI 视觉体系、并提升右侧控制台的商业仪表盘体验。

## 1. 优化目标

1. **去 Emoji 化**：彻底移除界面中的所有 Emoji，提升商业化工具的专业度与正式感。
2. **图谱标题与图例样式对齐**：
   - 标题“您的个人外贸知识拓扑网络 (已解锁报告)”缩短为“市场图谱”，字重从 `600` 降为 `500`（细体），颜色与筛选栏的“国家/市场”文字颜色一致。
   - “客户洞察”前置圆点颜色调整为图谱的橘红色（`#ff641e`），“品类分析”前置圆点颜色调整为图谱的灰色（`#7a756f`），两个文本的字体大小、颜色、粗细与“国家/市场”完全对齐。
3. **右侧看板（未选中报告时）**：
   - 设计 2x2 指标卡片网格，利用前端传入的 `nodes` 计算展示：已解锁报告总数、客户洞察报告数、品类分析报告数、覆盖市场/地区数。
   - 统一使用浅暖米沙色背景与细边框，数据数字使用橘红色作为视觉焦点，底部配以干净的操作提示文字。
4. **右侧详情（选中报告时）**：
   - 移除“报告 ID”。
   - 移除“所涉国家/市场”展示（它不属于图谱的关系边维度）。
   - 将所有 Tag 的彩色背景统一调整为柔和的暖米色背景 `rgba(160, 109, 68, 0.05)`，深灰色文字 `var(--color-text)` 与精致的细边框 `1px solid rgba(160, 109, 68, 0.15)`，消除五颜六色带来的不协调感。
   - 过滤逻辑：
     - 若为公司报告（`category !== 'product'`）：展示 4 层图谱涉及的关系（“经营玩家/品牌”、“竞争对手”、“涉及品类”、“覆盖渠道”）。
     - 若为品类报告（`category === 'product'`）：仅展示“涉及品类”的关系。
5. **底部报告标题**：“最近解锁的报告 (最多显示10篇)”简化为“你的报告”（无 Emoji 且缩短）。

## 2. 详细修改设计

### 2.1 组件 `components/ObsidianGraph.tsx`
- **标题文本与样式**：
  - 修改 `📂 您的个人外贸知识拓扑网络 (已解锁报告)` 为 `市场图谱`。
  - 将样式中的 `fontWeight: 600` 改为 `500`，`color: '#475569'` 改为 `var(--color-muted)`。
- **图例点样式**：
  - 将原来的 `● 客户洞察` 与 `● 品类分析` 字体改为细体，颜色为 `var(--color-muted)`，字号改为 `0.85rem`。
  - 圆点单独使用对应的图谱颜色包裹：客户洞察前缀圆点颜色为 `#ff641e`，品类分析前缀圆点颜色为 `#7a756f`。

### 2.2 组件 `components/NodeProfilePanel.tsx`
- **数据源扩展**：
  - props 新增可选参数 `allNodes?: GraphNode[]`。
- **未选中状态看板 (selectedNode === null)**：
  - 去除 `💡` Emoji。
  - 从 `allNodes` 中提取：
    - `totalReports`: `node_type === 'report'` 的节点数
    - `customerReports`: `node_type === 'report' && category !== 'product'` 的节点数
    - `productReports`: `node_type === 'report' && category === 'product'` 的节点数
    - `coveredMarkets`: 收集所有报告节点的 `market_region` 去重后的数量。
  - 2x2 网格卡片渲染：卡片样式参照现有 VI，背景为轻微的 `rgba(160, 109, 68, 0.03)`，细边框，字体为细体，大数字为 `#ff641e`，数据标签为 `var(--color-muted)`。
  - 下方保留无 Emoji 的提示词。
- **选中状态报告详情 (selectedNode.node_type === 'report')**：
  - 彻底去除“报告 ID: xxx”这一行渲染。
  - 去除“所涉国家/市场”渲染。
  - 根据 `selectedNode.category === 'product'` 区分：
    - 品类报告：仅渲染“涉及品类”标签列表、关联新表单、报告概述、以及阅读/删除动作。
    - 公司报告：渲染“经营玩家/品牌”、“竞争对手”、“涉及品类”、“覆盖渠道” 4 层关系。
  - 所有展示的实体 Tag 样式统一（以“经营玩家/品牌”为例）：
    - `background: 'rgba(160, 109, 68, 0.05)'`
    - `color: 'var(--color-text)'`
    - `border: '1px solid rgba(160, 109, 68, 0.15)'`
    - `fontSize: '0.75rem'`
- **其他细节**：
  - 彻底清除所有标题和按钮前的 Emoji，如“🏢 商业实体画像”改为“商业实体画像”，“🗑️ 永久删除此报告”改为“永久删除此报告”。

### 2.3 页面 `pages/my-graph.tsx`
- **右侧侧边栏调用**：
  - 给 `<NodeProfilePanel>` 组件传入 `allNodes={activeGraphData.nodes}`。
- **标题及底部更新**：
  - 移除页面内其他零碎 Emoji（如初始演示提示里的 Emoji、警告框里的 `⚠️` 等）。
  - 修改第 835 行的标题：`🔓 最近解锁的报告 (最多显示10篇)` 改为 `你的报告`。

## 3. 自我审查 (Specs Review)

- **占位符**：无，所有数据提取和逻辑分支均已写明。
- **一致性**：前台圆点颜色 `#ff641e`、`#7a756f` 与图谱核心物理渲染中的橙色、灰色完全一致。
- **作用域**：仅修改图谱主页与侧边栏、Legend 的视觉展示 and 状态交互，绝不破坏现有的报告解锁或实体删除等核心业务。
