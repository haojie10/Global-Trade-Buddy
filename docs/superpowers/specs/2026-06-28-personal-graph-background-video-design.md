# 个人图谱网页背景视频技术设计规格

本规格文件定义了基于 **Warm Cream UI** 风格的个人图谱网页背景视频（20秒无缝循环）的技术实现细节与设计参数。

---

## 1. 视觉设计系统

### 1.1 色彩与层级 (Warm Cream Theme)
背景视频应与 App 现有的 Warm Cream 视觉规范完全对齐，整体呈现低饱和度的温和水印质感，不喧宾夺主。

* **背景渐变 (Background)**：
  ```css
  background: 
    radial-gradient(circle at 10% 10%, rgba(255, 100, 30, 0.012) 0%, transparent 45%),
    radial-gradient(circle at 90% 90%, rgba(255, 100, 30, 0.008) 0%, transparent 45%),
    linear-gradient(135deg, #fdfbf7 0%, #f5f0eb 100%);
  ```
* **圆点样式 (Nodes)**：
  * **尺寸**：所有圆点尺寸不论颜色完全一致，半径统一设为 **`11.0px`**（直径 `22.0px`），带来类似散景/光斑的呼吸感。
  * **分类配色**：随机分配两色圆点（占比约为 1:1）：
    * **橘红色点**：`rgba(255, 100, 30, 0.22)`
    * **暗灰色点**：`rgba(122, 117, 111, 0.18)`
* **线条样式 (Lines)**：
  线宽增加一倍以保证轮廓线清晰，共有 4 种类型，基准透明度限制在 `[0.08, 0.25]` 区间（极远时淡出，但绝不消失以防分裂）：
  * **竞争关系线**：红色实线，颜色为 `rgba(211, 47, 47, opacity * 0.5)`，线宽 **`1.8px`**。
  * **供销关系线**：橘色流动虚线，颜色为 `rgba(255, 100, 30, opacity * 0.7)`，线宽 **`2.0px`**，虚线模式为 `[6, 6]`，向着连接点不断滚动流动。
  * **经营关系线**：蓝色实线，颜色为 `rgba(21, 101, 192, opacity * 0.5)`，线宽 **`1.8px`**。
  * **涉及关系线**：灰色点线，颜色为 `rgba(160, 155, 149, opacity * 0.4)`，线宽 **`1.4px`**，虚线模式为 `[3, 4]`。

### 1.2 无限画布与全局连通 (Infinite Canvas)
* **大范围分布**：圆点总数设为 **`40个`**，其基础坐标分布于 $[-0.15, 1.15]$ 的大区域内，部分点会自然漂移出屏幕边界，呈现无限画布延伸感。
* **全局相连**：底层拓扑结构使用**生成树算法 (Spanning Tree)** 初始化，再加上 30 条闭环随机边，在数学上绝对保证是一个单一的连通分量，没有任何孤立的单独板块。

## 2. 动效与循环算法 (20s Mathematical Loop)

为了在 HyperFrames 中达成 100% 完美的 20 秒首尾无缝循环，运动逻辑完全放弃随机物理引力，而采用确定性的简谐叠加运动方程。

### 2.1 节点慢速同步漂移方程
每个非选中状态节点的基础位置为 $(bx_k, by_k)$，在时间 $t$ 时的实际绘制坐标 $(x_k(t), y_k(t))$ 计算为：
$$x_k(t) = bx_k \cdot W + A_{k,1} \sin(2\pi f_{k,1} t + \phi_{k,1}) + A_{k,2} \sin(2\pi f_{k,2} t + \phi_{k,2})$$
$$y_k(t) = by_k \cdot H + B_{k,1} \cos(2\pi g_{k,1} t + \psi_{k,1}) + B_{k,2} \cos(2\pi g_{k,2} t + \psi_{k,2})$$

* **循环闭环条件**：所有的运动频率 $f_{k,j}$ 和 $g_{k,j}$ 都必须是 $1/20 = 0.05\text{ Hz}$ 的整数倍。
* **频率参数**（速度上调 50% 方案）：
  * $f_{k,1}$ 和 $g_{k,1}$ 从 $\{0.10\text{ Hz}, 0.15\text{ Hz}\}$ 中随机选择。
  * $f_{k,2}$ 和 $g_{k,2}$ 从 $\{0.15\text{ Hz}, 0.20\text{ Hz}\}$ 中随机选择。
* **漂移振幅**：
  * $A_{k,1}, B_{k,1}$ 介于 $7\text{px}$ 到 $15\text{px}$ 之间。
  * $A_{k,2}, B_{k,2}$ 介于 $2\text{px}$ 到 $6\text{px}$ 之间。

### 2.2 选中焦点 5 秒轮替与平滑 Cross-Fade
画面中任何时刻仅显示**一个**选中状态。在 20 秒中，通过特定位置的 4 个可见节点进行轮替：
* **节点分布**：
  * 节点 0（上方）：$(bx=0.5, by=0.25)$，在 $0\text{s} - 5\text{s}$ 选中（中心 $2.5\text{s}$）
  * 节点 1（右侧）：$(bx=0.76, by=0.5)$，在 $5\text{s} - 10\text{s}$ 选中（中心 $7.5\text{s}$）
  * 节点 2（下方）：$(bx=0.5, by=0.75)$，在 $10\text{s} - 15\text{s}$ 选中（中心 $12.5\text{s}$）
  * 节点 3（左侧）：$(bx=0.24, by=0.5)$，在 $15\text{s} - 20\text{s}$ 选中（中心 $17.5\text{s}$）
* **插值淡入淡出公式**：
  设第 $i$ 个选中点（$i \in [0, 3]$）的中心时间为 $T_c = i \cdot 5 + 2.5\text{s}$。在时间 $t$ 时的选中因子 $S_i(t)$ 为：
  $$\text{diff} = \min(|t - T_c|, 20 - |t - T_c|)$$
  $$S_i(t) = \begin{cases} 
    1.0 & \text{if } \text{diff} \le 2.0 \\
    0.0 & \text{if } \text{diff} \ge 3.0 \\
    1.0 - (\text{diff} - 2.0) & \text{if } 2.0 < \text{diff} < 3.0 
  \end{cases}$$
  此公式可在每次交接时提供 1 秒的完美 cross-fade，并且由于以 20 为模计算，首尾衔接绝对平滑。

### 2.3 选中状态 L 型四角呼吸样式
当节点 $k$ 的选中因子 $S_k(t) > 0$ 时，触发以下特殊绘制：
* **核心高亮**：节点填充色变为亮橘色并提供发光，核心透明度为 $0.25 + 0.50 \cdot S_k(t)$：
  `fillStyle = rgba(255, 100, 30, 0.25 + 0.50 * S_k(t))`。
* **发光阴影**：添加模糊半径为 $6 \cdot S_k(t)$ 的暖橘色外发光。
* **L 型四角框**：围绕圆点绘制收缩膨胀的正方形四个 L 角。
  * **大小呼吸曲线**：`boxSize = radius * 2.5 * pulse`，其中 `pulse = 1 + sin(t * Math.PI * 2 * 1.5) * 0.15`（呼吸频率 $1.5\text{ Hz}$）。
  * **四角线条透明度**：正比于 $S_k(t)$ 的淡入淡出，即 `strokeStyle = rgba(255, 100, 30, 0.65 * S_k(t))`，线宽 $1.5\text{px}$。
  * **绘制坐标**：在圆点外部坐标 $(x \pm \text{half}, y \pm \text{half})$ 处向中心分别延伸 $\text{len} = \text{boxSize}/4$ 的直线。

### 2.4 线段流动速度
* **供销关系流动速度**：`lineDashOffset = -t * 36`（每秒滚动 $36\text{px}$，在 20 秒时流动位移为 $720\text{px}$，为短划虚线周期 $12\text{px}$ 的整数倍，实现完美无缝流动循环）。

## 3. 实现与验证方案

### 3.1 代码部署与环境
1. 在项目目录新建 `components/BackgroundGraph.tsx` 作为可在 App/网页中随时引用的实时网页背景 Canvas。
2. 新建 `components/BackgroundGraph.module.css` 声明 `fixed` 全屏置底层级 (`z-index: -1`)。

### 3.2 20秒无缝循环 MP4 背景视频导出 (HyperFrames)
使用 HyperFrames 录制 Canvas 画面，导出 20 秒循环 MP4：
```bash
# 初始化 HyperFrames 环境（若未初始化）
npx hyperframes init

# 渲染 20s 无缝循环背景视频为 MP4
npx hyperframes render --entry=components/BackgroundGraph.tsx --duration=20 --fps=30 --output=public/videos/graph-background.mp4
```

### 3.3 自动化验证
* 首尾帧一致性校对：使用 SSIM 验证第 0 帧与第 600 帧（30fps 下 20秒末尾）的图像差异值，SSIM 应接近 1.00，以证明无缝闭环成立。
